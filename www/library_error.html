Library Linking Error

The native interface classes rely on library dlls. These dlls must be available to the virtual machine at runtime, and require the library path to be set accordingly, which can be achieved with the Java option: 

     -Djava.library.path="C:/download/path/bhavaya/lib"

where download/path should be changed to reflect the actual location of the library.

For example, you might run the program as:

     java -cp ".;c:/Projects/bhavaya/bin/bhavaya.jar"
          -Djava.library.path="C:/Projects/bhavaya/lib" ThreadDumpDemo

If this variable is not set correctly then an error similar to the following will result:
