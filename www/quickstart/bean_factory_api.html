The BeanFactory API

BeanFactory extends java.util.Map (by implementing org.bhavaya.collection.BeanMap) thus all of Map's methods are available. Just those methods most likely to be used and those specific to BeanFactory are described below.

method return type description
getInstance(Class type) BeanFactory Returns an instance of a BeanFactory for a given type and default dataSource. All BeanFactories are created by the appropriate definitions in schema.xml.
getInstance(Class type, String dataSourceName) BeanFactory Returns an instance of a BeanFactory for a given type and dataSource. All BeanFactories are created by the appropriate definitions in schema.xml.
get(Object key) Object Returns the bean with the given key. The key used must be of an instance with compatible equals and hashCode as the key that will be generated by BeanFactory. For example, if a bean is identified by the number "1233353", it is important whether this is an Integer, Double, BigDecimal, String etc. BeanFactory decides the type of the key based on the type of the column in the database table. Where a bean is identified by a compound key, the key must be an ArrayList of key component values. The values must be ordered alphabetically by column name.
get(Object key, String indexname) Object Returns the object matching the key, uses the indexing strategy indexname when finding the object.
newBeanInstance() Object Returns a new bean, of type equal to that of this BeanFactory's instance.
getAllObjects() Object[] Returns all objects
getAllBeanCollection() CriteriaBeanCollection Returns all objects from the database as a CriteriaBeanCollection
getBeanCollection(CriterionGroup criterionGroup) CriteriaBeanCollection Returns all objects matching the criterionGroup as a CriteriaBeanCollection
getObjects(CriterionGroup criterionGroup) Object[] Returns all objects matching the criterionGroup as an Object[]
getObjects(Object[] keys) Object[] Returns an object set for the given keyset keys.
getObjects(Object[] keys, String indexname) Object[] Returns the object[] matching the key array keys, uses the indexing strategy indexname when finding the objects.
clearAllBeanFactories(String dataSourceName, Class beanFactoryType, boolean clearLowVolatilityBeanFactories, boolean fireEvents) void (static) Clear all BeanFactories that use dataSource. Typically, this is called by clients using a transactional dataSource, i.e. a dataSource for which autoCommit is false.
clearAllBeanFactories(boolean clearLowVolatilityBeanFactories, boolean fireEvents) void (static) Clear all BeanFactories.
