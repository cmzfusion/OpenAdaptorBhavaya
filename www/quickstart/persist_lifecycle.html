The Persistence Layer - The LifeCycle Interface

By implementing the org.bhavaya.beans.LifeCycle interface a bean can ensure that specific actions take place (methods are called) when certain database events occur. The methods that are called and when they are triggered are described in the table below.

Method Trigger
public void init() BeanFactory calls this at the end of a transaction if it has instantiated the bean.
public void inserted() BeanFactory calls this at the end of a transaction if the bean is due to an "insert" sql statement.
public void updated() BeanFactory calls this at the end of a transaction if it has updated the bean due to an "update" sql statement.
public void deleted() BeanFactory calls this at the end of a transaction if it has removed the bean due to a "delete" sql statement.

For example, Bond.init() could increment a private static variable bondCount. NB these methods will only be called if the class explicitly implements the LifeCycle interface.
