CPU Load

The CpuLoad class (org.bhavaya.util) provides load information for the system's processor(s). The class uses a native c compiled program to access the machine's resources.

The class can be seen in action by running the demo directly using JavaWebStart, from the source code, or if you have downloaded and extracted examples.zip it can be run with: ant cpuloaddemo. A screen shot of the CPU Load Demo is shown below.

CpuLoad is a singleton class, with two methods for accessing load information:

     System.out.println("Load = " + CpuLoad.getInstance().getLoad()*100.0 + "%");
     System.out.println("Average Load = " + CpuLoad.getInstance().getLoadAverage()*100 + "%");

When using this component, it is necessary to ensure that the VM knows the location of the native library, by using an appropriately modified command line argument (see library error) similar to:

     -Djava.library.path="C:/extract/location/bhavaya/lib"

(Also remember to add Bhavaya's jar files to your classpath (extract_location/bhavaya/bin/bhavaya.jar and extract_location/bhavaya/lib/log4j-1.2.15-patched.jar).)
