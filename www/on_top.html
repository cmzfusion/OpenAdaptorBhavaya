Always on Top

The NativeWindow class (org.bhavaya.ui.NativeWindow) allows a frame to be set (or unset) to be always on top. An instance of the class is obtained for each frame to be controlled. Adjustments to a frame's on-top state should only be made from the Event Dispatch Thread with a command similar to:

     NativeWindow.getInstance(frame).setAlwaysOnTop(true);

There is a demo available via JavaWebStart, or if you have downloaded and extracted examples.zip it can be run with: ant ontopdemo. The source code for an example usage is also available and its API is given in the table below.

Function Return type Usage
isLibraryLoaded boolean reports whether the external native library was successfully loaded
getInstance(Frame) NativeWindow returns an instance of the NativeWindow class for use with the supplied Frame
setAlwaysOnTop(boolean) void set boolean to true to cause the window to be always on top, false otherwise
isAlwaysOnTop() boolean returns true if the window is set to be always on top, false otherwise

When using this component, it is necessary to ensure that the VM knows the location of the native library, by using an appropriately modified command line argument (see library error) similar to:

     
     -Djava.library.path="C:/extract/location/bhavaya/lib"

(Also remember to add Bhavaya's jar files to your classpath (extract_location/bhavaya/bin/bhavaya.jar and extract_location/bhavaya/lib/log4j-1.2.15-patched.jar).)
