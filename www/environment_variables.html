Environment Variables

Access to system environment variables is provided by Bhavaya's singleton class Environment (org.bhavaya.util.Environment), which uses a c compiled native library. Its use is as simple as:

      System.out.println(Environment.getProperty("USERNAME"));

The getProperty method returns a string, and takes as its argument the name of the environment variable required. The function will return null if the specified variable does not exist. There is a demo available via JavaWebStart.

The table below summarises the environment variable access available from the Environment class.

Function Return type Usage
getProperty(String property) String Returns the value of the property
getProperties() Map Returns a property map (LinkedHashMap), where the key used to obtain the value of each property is the property's name as an all upper-case string.

When using this component, it is necessary to ensure that the VM knows the location of the native library, by using an appropriately modified command line argument (see library error) similar to:

     -Djava.library.path="C:/extract/location/bhavaya/lib"

(Also remember to add Bhavaya's jar files to your classpath (extract_location/bhavaya/bin/bhavaya.jar and extract_location/bhavaya/lib/log4j-1.2.15-patched.jar).)
